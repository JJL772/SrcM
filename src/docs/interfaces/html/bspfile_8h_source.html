156 // ------------------------------------------------------------------------------------------------ //
157 // Displacement neighbor rules
158 // ------------------------------------------------------------------------------------------------ //
159 //
160 // Each displacement is considered to be in its own space:
161 //
162 // NEIGHBOREDGE_TOP
163 //
164 // 1 --- 2
165 // | |
166 // NEIGHBOREDGE_LEFT | | NEIGHBOREDGE_RIGHT
167 // | |
168 // 0 --- 3
169 //
170 // NEIGHBOREDGE_BOTTOM
171 //
172 //
173 // Edge edge of a displacement can have up to two neighbors. If it only has one neighbor
174 // and the neighbor fills the edge, then SubNeighbor 0 uses CORNER_TO_CORNER (and SubNeighbor 1
175 // is undefined).
176 //
177 // CORNER_TO_MIDPOINT means that it spans [bottom edge,midpoint] or [left edge,midpoint] depending
178 // on which edge you're on.
179 //
180 // MIDPOINT_TO_CORNER means that it spans [midpoint,top edge] or [midpoint,right edge] depending
181 // on which edge you're on.
182 //
183 // Here's an illustration (where C2M=CORNER_TO_MIDPOINT and M2C=MIDPOINT_TO_CORNER
184 //
185 //
186 // C2M M2C
187 //
188 // 1 --------------> x --------------> 2
189 //
190 // ^ ^
191 // | |
192 // | |
193 // M2C | | M2C
194 // | |
195 // | |
196 //
197 // x x x
198 //
199 // ^ ^
200 // | |
201 // | |
202 // C2M | | C2M
203 // | |
204 // | |
205 //
206 // 0 --------------> x --------------> 3
207 //
208 // C2M M2C
209 //
210 //
211 // The CHILDNODE_ defines can be used to refer to a node's child nodes (this is for when you're
212 // recursing into the node tree inside a displacement):
213 //
214 // ---------
215 // | | |
216 // | 1 | 0 |
217 // | | |
218 // |---x---|
219 // | | |
220 // | 2 | 3 |
221 // | | |
222 // ---------
223 //
224 // ------------------------------------------------------------------------------------------------ //
