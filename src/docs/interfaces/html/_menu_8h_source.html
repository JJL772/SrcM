27 //-----------------------------------------------------------------------------
28 // Purpose: A menu is a list of items that can be selected with one click, navigated
29 // with arrow keys and/or hot keys, and have a lit behavior when mouse over.
30 // It is NOT the button which opens the menu, but only the menu itself.
31 //
32 // Behaviour spec:
33 // Menu navigation can be done in 2 modes, via keyboard keys and via mouse.
34 // Clicking on menu button opens menu.
35 // Only one item in a menu is highlighted at a time.
36 // Only one submenu in a menu is open at a time.
37 // Disabled menuitems get highlighted via mouse and keys but will not activate.
38 //
39 // Mouse:
40 // Moving mouse into a menuitem highlights it.
41 // If the menuitem has a cascading menu, the menu opens when the mouse enters
42 // the menuitem. The cascading menuitem stays highlighted while its menu is open.
43 // No submenu items are highlighted by default.
44 // Moving the mouse into another menuitem closes any previously open submenus in the list.
45 // Clicking once in the menu item activates the menu item and closes all menus.
46 // Moving the mouse off a menuitem unhighlights it.
47 // The scroll bar arrows can be used to move up/down the menu one item at a time.
48 // The clicking and dragging on the scroll bar nob also scrolls the menu items.
49 // If a highlighed menuitem scrolls off, and the user then begins navigating via keys,
50 // the menu will snap the scroll bar so the highlighted item is visible.
51 // If user has been navigating via keys, moving the mouse over a menu item
52 // highlights it.
53 // Mousewheel:
54 // You must have the mouse inside the menu/scroll bar to use the wheel.
55 // The mouse wheel moves the highlighted menuitem up or down the list.
56 // If the list has no scroll bar the wheel will cycle from the bottom of the list
57 // to the top of the list and vice versa.
58 // If the list has a scrollbar the mouse wheel will stop at the top or bottom
59 // of the list.
60 // If the mouse is over the scroll bar no items are highlighted.
61 // Keyboard:
62 // When a menu is opened, no items are highlighted.
63 // If a menuitem has a cascading menu it does not open when the item is highlighted.
64 // The down arrow selects the next item in the list.
65 // (first item if none are highlighted and there is a scrollbar).
66 // The up arrow selects the previous item in the list
67 // (first item if none are highlighted and there is a scrollbar, last item if none are
68 // highlighted and there is no scrollbar).
69 // Selecting a new menuitem closes any previously open submenus in the list.
70 // The enter key activates the selected item and closes all menus.
71 // If the selected item has a cascading menu, activating it opens its submenu.
72 // These may also be activated by pressing the right arrow.
73 // Pressing the left arrow closes the submenu.
74 // When the submenu is opened the cascading menuitem stays highlighted.
75 // No items in the submenu are highlighted when it is opened.
76 //
77 // Note: Cascading menuitems in menus with a scrollbar is not supported.
78 // Its a clunky UI and if we want this we should design a better solution,
79 // perhaps along the lines of how explorer's bookmarks does it.
80 // It currently functions, but there are some arm/disarm bugs.
81 //
82 //
