56 // NOTE: DataTable receive proxies work differently than the other proxies.
57 // pData points at the object + the recv table's offset.
58 // pOut should be set to the location of the object to unpack the data table into.
59 // If the parent object just contains the child object, the default proxy just does *pOut = pData.
60 // If the parent object points at the child object, you need to dereference the pointer here.
61 // NOTE: don't ever return null from a DataTable receive proxy function. Bad things will happen.
62 typedef void (*DataTableRecvVarProxyFn)(const RecvProp *pProp, void **pOut, void *pData, int objectID);
