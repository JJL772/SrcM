401  /* the filter operation codes that go in the key part of MatchMakingKeyValuePair_t should be one of these:
402 
403  "map"
404  - Server passes the filter if the server is playing the specified map.
405  "gamedataand"
406  - Server passes the filter if the server's game data (ISteamGameServer::SetGameData) contains all of the
407  specified strings. The value field is a comma-delimited list of strings to match.
408  "gamedataor"
409  - Server passes the filter if the server's game data (ISteamGameServer::SetGameData) contains at least one of the
410  specified strings. The value field is a comma-delimited list of strings to match.
411  "gamedatanor"
412  - Server passes the filter if the server's game data (ISteamGameServer::SetGameData) does not contain any
413  of the specified strings. The value field is a comma-delimited list of strings to check.
414  "gametagsand"
415  - Server passes the filter if the server's game tags (ISteamGameServer::SetGameTags) contains all
416  of the specified strings. The value field is a comma-delimited list of strings to check.
417  "gametagsnor"
418  - Server passes the filter if the server's game tags (ISteamGameServer::SetGameTags) does not contain any
419  of the specified strings. The value field is a comma-delimited list of strings to check.
420  "and" (x1 && x2 && ... && xn)
421  "or" (x1 || x2 || ... || xn)
422  "nand" !(x1 && x2 && ... && xn)
423  "nor" !(x1 || x2 || ... || xn)
424  - Performs Boolean operation on the following filters. The operand to this filter specifies
425  the "size" of the Boolean inputs to the operation, in Key/value pairs. (The keyvalue
426  pairs must immediately follow, i.e. this is a prefix logical operator notation.)
427  In the simplest case where Boolean expressions are not nested, this is simply
428  the number of operands.
429 
430  For example, to match servers on a particular map or with a particular tag, would would
431  use these filters.
432 
433  ( server.map == "cp_dustbowl" || server.gametags.contains("payload") )
434  "or", "2"
435  "map", "cp_dustbowl"
436  "gametagsand", "payload"
437 
438  If logical inputs are nested, then the operand specifies the size of the entire
439  "length" of its operands, not the number of immediate children.
440 
441  ( server.map == "cp_dustbowl" || ( server.gametags.contains("payload") && !server.gametags.contains("payloadrace") ) )
442  "or", "4"
443  "map", "cp_dustbowl"
444  "and", "2"
445  "gametagsand", "payload"
446  "gametagsnor", "payloadrace"
447 
448  Unary NOT can be achieved using either "nand" or "nor" with a single operand.
449 
450  "addr"
451  - Server passes the filter if the server's query address matches the specified IP or IP:port.
452  "gameaddr"
453  - Server passes the filter if the server's game address matches the specified IP or IP:port.
454 
455  The following filter operations ignore the "value" part of MatchMakingKeyValuePair_t
456 
457  "dedicated"
458  - Server passes the filter if it passed true to SetDedicatedServer.
459  "secure"
460  - Server passes the filter if the server is VAC-enabled.
461  "notfull"
462  - Server passes the filter if the player count is less than the reported max player count.
463  "hasplayers"
464  - Server passes the filter if the player count is greater than zero.
465  "noplayers"
466  - Server passes the filter if it doesn't have any players.
467  "linux"
468  - Server passes the filter if it's a linux server
