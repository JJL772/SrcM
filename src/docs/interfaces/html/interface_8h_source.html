9 // This header defines the interface convention used in the valve engine.
10 // To make an interface and expose it:
11 // 1. The interface must be ALL pure virtuals, and have no data members.
12 // 2. Define a name for it.
13 // 3. In its implementation file, use EXPOSE_INTERFACE or EXPOSE_SINGLE_INTERFACE.
14 
15 // Versioning
16 // There are two versioning cases that are handled by this:
17 // 1. You add functions to the end of an interface, so it is binary compatible with the previous interface. In this case,
18 // you need two EXPOSE_INTERFACEs: one to expose your class as the old interface and one to expose it as the new interface.
19 // 2. You update an interface so it's not compatible anymore (but you still want to be able to expose the old interface
20 // for legacy code). In this case, you need to make a new version name for your new interface, and make a wrapper interface and
21 // expose it for the old interface.
22 
23 // Static Linking:
24 // Must mimic unique seperate class 'InterfaceReg' constructors per subsystem.
25 // Each subsystem can then import and export interfaces as expected.
26 // This is achieved through unique namespacing 'InterfaceReg' via symbol _SUBSYSTEM.
27 // Static Linking also needs to generate unique symbols per interface so as to
28 // provide a 'stitching' method whereby these interface symbols can be referenced
29 // via the lib's primary module (usually the lib's interface exposure)
30 // therby stitching all of that lib's code/data together for eventual final exe link inclusion.

85 // Use this to expose an interface that can have multiple instances.
86 // e.g.:
87 // EXPOSE_INTERFACE( CInterfaceImp, IInterface, "MyInterface001" )
88 // This will expose a class called CInterfaceImp that implements IInterface (a pure class)
89 // clients can receive a pointer to this class by calling CreateInterface( "MyInterface001" )
90 //
91 // In practice, the shared header file defines the interface (IInterface) and version name ("MyInterface001")
92 // so that each component can use these names/vtables to communicate
93 //
94 // A single class can support multiple interfaces through multiple inheritance
95 //
96 // Use this if you want to write the factory function.
97 #if !defined(_STATIC_LINKED) || !defined(_SUBSYSTEM)
98 #define EXPOSE_INTERFACE_FN(functionName, interfaceName, versionName) \
99  static InterfaceReg __g_Create##interfaceName##_reg(functionName, versionName);
100 #else
101 #define EXPOSE_INTERFACE_FN(functionName, interfaceName, versionName) \
102  namespace _SUBSYSTEM \
103  { \
104  static InterfaceReg __g_Create##interfaceName##_reg(functionName, versionName); \
105  }
106 #endif
