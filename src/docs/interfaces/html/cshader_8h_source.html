197  // regarding the macro above: the "::" was added to the first argument in order to disambiguate it for GCC.
198  // for example, in cloak.cpp, this usage appears:
199  // SHADER_PARAM_OVERRIDE( COLOR, SHADER_PARAM_TYPE_COLOR, "{255 255 255}", "unused", SHADER_PARAM_NOT_EDITABLE )
200  // which in turn tries to ask the compiler to instantiate an object like so:
201  // static CShaderParam COLOR( (ShaderMaterialVars_t)COLOR, SHADER_PARAM_TYPE_COLOR, "{255 255 255}", "unused", SHADER_PARAM_NOT_EDITABLE )
202  // and GCC thinks that the reference to COLOR in the arg list is actually a reference to the object we're in the middle of making.
203  // and you get --> error: invalid cast from type ‘Cloak_DX90::CShaderParam’ to type ‘ShaderMaterialVars_t’
204  // Resolved: add the "::" so compiler knows that reference is to the enum, not to the name of the object being made.
