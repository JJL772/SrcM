63  // NOTE FOR ENTITY WRITERS:
64  //
65  // In 90% of the cases, you should hook OnPreDataChanged/OnDataChanged instead of

67  //
68  // The DataChanged events are only called once per frame whereas Pre/PostDataUpdate

70  //
71  // OnDataChanged is called during simulation where entity origins are correct and
72  // attachments can be used. whereas PostDataUpdate is called while parsing packets
73  // so attachments and other entity origins may not be valid yet.
74  //
