29 /*
30 
31 The GameEventManager keeps track and fires of all global game events. Game events
32 are fired by game.dll for events like player death or team wins. Each event has a
33 unique name and comes with a KeyValue structure providing informations about this
34 event. Some events are generated also by the engine.
35 
36 Events are networked to connected clients and invoked there to. Therefore you
37 have to specify all data fields and there data types in an public resource
38 file which is parsed by server and broadcasted to it's clients. A typical game
39 event is defined like this:
40 
41  "game_start" // a new game starts
42  {
43  "roundslimit" "long" // max round
44  "timelimit" "long" // time limit
45  "fraglimit" "long" // frag limit
46  "objective" "string" // round objective
47  }
48 
49 All events must have unique names (case sensitive) and may have a list
50 of data fields. each data field must specify a data type, so the engine
51 knows how to serialize/unserialize that event for network transmission.
52 Valid data types are string, float, long, short, byte & bool. If a
53 data field should not be broadcasted to clients, use the type "local".
54 */
