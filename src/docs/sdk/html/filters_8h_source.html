3 // Purpose: Filters are outboard entities that hold a set of rules that other
4 // entities can use to determine behaviors.
5 //
6 // For example, triggers can use an activator filter to determine who
7 // activates them. NPCs and breakables can use a damage filter to
8 // determine what can damage them.
9 //
10 // Current filter criteria are:
11 //
12 // Activator name
13 // Activator class
14 // Activator team
15 // Damage type (for damage filters only)
16 //
17 // More than one filter can be combined to create a more complex boolean
18 // expression by using filter_multi.
19 //
